# Проект "GIS Expert / IT Hub Uralsk"

Проект экосистемы IT Hub Uralsk, включающий образовательные курсы и ИТ-сообщество.

## Ключевые компоненты

### 1. Статическая генерация контента (SSG)
Сайт построен на базе модульной архитектуры с использованием Markdown для контента и HTML-шаблонов для макета.
- **Модули:** `community` (сообщество) и `course` (курсы).
- **Контент:** Хранится в `src/{module}/ui/content/*.md`.
- **Шаблоны:** Находятся в `src/{module}/ui/*.template.html`.
- **Сборка:** Кастомный скрипт `build.ts` на базе Bun и `markdown-it`.

### 2. Система аналитики
Собственная система сбора метрик без сторонних трекеров.
- **Трекер:** `src/app/ui/tracker.ts` (время нахождения, скролл, просмотр секций).
- **Бэкенд:** `src/app/server.ts` на Bun.serve.
- **БД:** SQLite (`course.sqlite`).

## Быстрый старт

1. **Установка:**
   ```bash
   bun install
   ```

2. **Разработка:**
   ```bash
   # Сборка контента и запуск сервера
   bun run start:dev
   ```

3. **Сборка (Production):**
   ```bash
   bun run build:prod
   ```

## Мониторинг и Аналитика

Для проверки работы метрик и просмотра заявок используйте встроенные утилиты:

- **Просмотр событий (метрики):**
  ```bash
  bun run metrics          # Последние 15 событий
  bun run metrics:watch    # Живой мониторинг (обновление каждые 2с)
  ```

- **Просмотр заявок:**
  ```bash
  bun run src/app/db-utils/view-submissions.ts
  ```

## Работа с контентом

Чтобы изменить текст на сайте:
1. Найдите нужный `.md` файл в `src/community/ui/content/` или `src/course/ui/content/`.
2. Отредактируйте его. Вы можете использовать специальные атрибуты для стилизации: `{.class #id}`.
3. Запустите сборку: `bun run build:dev`.

## Технологический стек
- **Runtime:** Bun
- **SSG:** Markdown-it + HTML Templates
- **Styling:** Vanilla CSS + W3.CSS
- **Database:** SQLite
- **Infrastructure:** PM2 for Production

## Работа с базой данных

### Запуск миграций
Перед обновлением схемы БД на продакшене:
1. `bun run db:backup` — создать бэкап.
2. `bun run db:migrate` — применить миграции.
3. `bun run db:status` — проверить результат.
